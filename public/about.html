About the Juxly FHIR Browser

It's all about being open.

The Juxly FHIR Browser exists to serve two primary purposes.

First it creates a web accessible user interface for non-developers to interact with a FHIR server and gain insight into the data provided. This is useful to functional testers and product designers in organizations who are using FHIR or hope to do so.

Secondly, the project is designed to use more simplistic technologies to open the door to more developers. Not everyone building web applications understands sophisticated frameworks like AngularJS or React. We felt it would be helpful to deliver FHIR usage examples based on the simplest of approaches - namely jQuery and Bootstrap.

Our hope is that this project/application will be useful in the situations we've described above. This is by no means a production oriented application. This is not how we at Juxly build our FHIR based applications. However, we do feel this application accurately demonstrates the basic interactions that make FHIR an amazing new tool in the world of healthcare IT.

Good luck, and have fun!
